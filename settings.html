                    World Progression
                    

                

                

                    

                        

                            World Progression automatically generates off-screen world simulation reports at a fixed in-world time interval.
                            Reports are written by a standalone LLM pass and stored in the {prefix}_World lorebook.
                            The trigger is deterministic — JavaScript checks the [TIME] block after every State Tracker update;
                            the AI only writes the report, never decides whether to generate one.
                        


                        
                        

                            

                                
                                Quick Start Guide
                                

                            

                            

                                
Creating a World Skeleton is optional but recommended — it gives World Progression content to work with (for Focus Randomization and generally), making the world bigger beyond the player. Without it, reports tend to stay limited to entities in the player's immediate bubble.

                                
    
                                    
  1. Create the Atmosphere Summary: Either write it manually or click Auto-Generate to grab it from your recent chat story (if one exists already). This establishes the setting's theme.
    2. 
                                    
  2. Generate the Skeleton: Click Generate Skeleton. This creates factions, locations, NPCs, and conflicts to simulate off-screen background activity, creating a Day 0 baseline. Remember that you can also add to the Skeleton Lorebook (after generation) yourself or edit the generated entities from the native SillyTavern Lorebook UI. This way, you can even have a fully granularly customized World Skeleton to base the World Reports on.
    3. 
                                    
  3. Set up Focus Randomization (Optional but Recommended): Enable this to force diversity in report coverage, rather than focusing only on the player's immediate bubble. Without Focus Randomization, the World Report may end up repeatedly covering the same threads because it has nothing forcing it to deviate. Not necessarily bad per se, but Focus Randomization guarantees a variety of entities are simulated in subsequent World Reports. It turns it into a “lottery.”
    4. 
                                    
  4. Generate the First Report: Trigger a report immediately using the Generate Now button, or let it fire automatically as the campaign progresses. If you run it later, it will incorporate entities that have organically accumulated in your adventure up to that point. Immediate generation is Skeleton-only.
    5. 
                                

                                

                                    
Miscellaneous

                                    
    
                                        
  • Remember that benched party members can also become part of World Reports. For example, if you leave a party member in the base as [BENCHED], World Progression may generate background activity for them.
    • 
                                        
  • 24h seems to be a decent interval for World Report auto-generation, but you are encouraged to experiment with shorter intervals (or longer, for that matter).
    • 
                                        
  • You can always also trigger World Progression manually whenever you like.
    • 
                                        
  • Injection Position affects the prominence of World Reports (adjustable from Context Injection Settings below). The lower the report is positioned (closer to the most recent message), the more prominent the report will be to the narrator.
